## 1. 문제 상황

>

* **상황:** 기프티콘 관리 페이지에서 수량이나 활성화 버튼을 누를시 작동되는 화면의 시간이 300ms로 느리게 보임
* **영향: 시각적으로 불편하게 느껴짐.**



## 2. 원인 분석

>

* **원인:**  화면을 생성하기 전에 서버로 값을 전달하고 받아오는 과정을 먼저 진행하기 때문이다.



## 3. 해결 과정 및 의사결정

>

1. **1차 시도:** 화면 랜더링과 서버 요청 로직의 순서를 바꾸었다.연속 클릭에 대한 무결성 문제가 발생함.
2. **2차 문제 발생** : 시각적 딜레이 문제는 해결됬지만 연속 클릭에 대한 무결성 문제가 제기됨!
3. **논의 :** 선생님께 질문을 하여 해당  연속 클릭에 대해 간격을 시간단위로 설정하여 방지할수 있는 방법이 있다는 것을 알게됨.



## 4. 최종 해결책

>

*  createShopSlice.js에서 화면 렌더링 로직(앞)과 서버 요청 로직(뒤)의 순서를 바꾸고  마지막 값만 서버에 반영하는 기능을 그 사이에 구현하여 해결함.



## 5. 배운 점 및 인사이트

>*이 이슈를 통해 무엇을 배웠나요? 앞으로는 어떻게 예방할까요? (가장 중요한 부분)*

* **Keep:** 문제가 생기자마자 의견을 나누며 공유해서 빠르게 해결책을 구상하는 점이 좋았음
* **Try:** 다음에는 프론트 로직을 작성할때 화면의 렌더링과 서버 데이터 흐름의 순서를 디테일하게 살펴보아야겠음.
* **Note:** 예약된 `setTimeout` 실행을 취소하는 함수에 대해 알게되었음.(이 함수를 이용해 연속 클릭을 했을떄 마지막 클릭을 제외한 나머지 클릭에 대해 무시하게 함)
