# RDBMS (Relational Database Management System)

## 1. 개요 (Overview)
### 1.1 정의
**RDBMS**는 **R**elational **D**ata**B**ase **M**anagement **S**ystem의 약자입니다.
데이터를 **행(Row)**과 **열(Column)**로 이루어진 **테이블(Table)** 형태(2차원 표)로 저장하며, 테이블 간의 **상호 관계**를 정의하여 데이터를 관리하는 시스템입니다.

### 1.2 비유: "스마트한 엑셀(Excel)"
엑셀 파일 여러 개(시트)를 만들고, '사용자 시트'의 ID와 '주문 시트'의 사용자ID를 서로 연결해둔 것과 비슷합니다. 다만 엑셀보다 훨씬 엄격하고, 빠르고, 대용량 처리가 가능합니다.

## 2. 핵심 특징 (Core Features)

### 2.1 테이블(Table) 구조
데이터는 무조건 정해진 표(Table) 형식에 맞춰 저장되어야 합니다.
-   **Schema (스키마):** 테이블의 구조(이름, 데이터 타입 등)를 미리 정의한 설계도입니다. 이 규칙에 맞지 않는 데이터는 저장될 수 없습니다. (엄격함)

### 2.2 관계 (Relationship)
RDBMS의 가장 큰 무기입니다. 데이터 중복을 피하기 위해 정보를 여러 테이블로 쪼개고, **키(Key)**를 통해 연결합니다.
-   **Primary Key (기본키):** 각 행을 구분하는 유일한 식별자 (예: 학번, 사원번호).
-   **Foreign Key (외래키):** 다른 테이블의 기본키를 참조하여 관계를 맺는 열.

### 2.3 SQL (Structured Query Language)
RDBMS와 대화하기 위한 표준 언어입니다. 오라클을 쓰든 MySQL을 쓰든 `SELECT`, `INSERT`, `UPDATE`, `DELETE`라는 기본 문법은 거의 동일합니다.

## 3. ACID 원칙
RDBMS가 데이터를 안전하게 보장하는 4가지 성질입니다. 은행 시스템 같은 곳에서 RDBMS를 쓰는 이유이기도 합니다.

| 성질 | 의미 | 설명 |
| :--- | :--- | :--- |
| **A**tomicity (원자성) | **All or Nothing** | 트랜잭션 내의 작업은 모두 성공하거나, 하나라도 실패하면 모두 취소(Rollback)되어야 한다. |
| **C**onsistency (일관성) | **규칙 준수** | 데이터는 미리 정의된 규칙(제약 조건)을 위반해서는 안 된다. (예: 잔액은 마이너스가 될 수 없음) |
| **I**solation (격리성) | **끼어들기 금지** | 하나의 트랜잭션이 실행되는 동안 다른 트랜잭션이 끼어들 수 없다. |
| **D**urability (지속성) | **영구 저장** | 성공적으로 완료된 트랜잭션 결과는 시스템이 꺼져도 영구적으로 반영되어야 한다. |

## 4. RDBMS vs NoSQL 비교

| 구분 | RDBMS (SQL) | NoSQL (Not Only SQL) |
| :--- | :--- | :--- |
| **데이터 구조** | 정해진 스키마(표)에 엄격하게 저장 | 스키마 없이 자유로운 형태(JSON 등)로 저장 |
| **관계** | 테이블 간 관계(Join) 중시 | 관계 정의가 없거나 느슨함 |
| **장점** | **데이터 무결성(신뢰성) 보장**, 복잡한 조회(Join) 유리 | **확장성**이 좋고, 읽기/쓰기 속도가 매우 빠름 |
| **단점** | 스케일 아웃(서버 확장)이 어렵고 유연성이 떨어짐 | 데이터 정합성 보장이 RDBMS보다 약할 수 있음 |
| **대표 제품** | Oracle, MySQL, PostgreSQL, MariaDB | MongoDB, Redis, Cassandra |

## 5. 대표적인 RDBMS 제품

1.  **Oracle:**
    -   강력한 성능과 안정성으로 대기업, 금융권, 공공기관(SI)에서 주로 사용합니다. (현재 진행 중인 팀 프로젝트나 국비 과정에서 주로 다루는 DB일 것입니다.)
2.  **MySQL / MariaDB:**
    -   오픈소스이며 무료입니다. 스타트업이나 중소규모 웹 서비스에서 가장 많이 사용합니다.
3.  **PostgreSQL:**
    -   최근 개발자들 사이에서 선호도가 급상승 중입니다. 표준 SQL을 잘 따르고 다양한 기능을 제공합니다.

---

## 6. 결론 (Conclusion)
RDBMS는 **"데이터의 무결성과 신뢰성이 중요한 서비스"**의 기본입니다.

쇼핑몰의 주문 내역, 은행의 계좌 정보, 사원 관리 시스템 등 **'정확도'가 생명인 데이터**는 반드시 RDBMS에 저장합니다. 백엔드 개발자로서 테이블을 설계하고(모델링), SQL을 통해 데이터를 자유자재로 다루는 능력은 필수입니다.
