## Topic 객체지향언어란 무엇이고 왜 JAVA가 객체지향언어인지 설명해주세요. 2025-12-02

- 하나의 주제를 정하고 단순한 정리가 아닌 개념적으로 학습한 내용을 토대로 정리하고 학습한다.

### **Why (왜 사용하는가? 왜 중요한가?)**

> 이 개념이 실무, 설계, 면접에서 중요한 이유를 정리한다.

- **실무**: 객체지향 개념이 없다면 코드가 절차적으로 길게 늘어져, 기능 추가나 수정 시 스파게티 코드(Spaghetti Code)가 되어 유지보수가 불가능해진다.
- **구조적 의미**: 데이터와 기능을 '객체'라는 단위로 묶어(캡슐화) 모듈화하고, 부품을 조립하듯 개발하여 코드의 재사용성과 확장성을 극대화한다.
- **면접 의도**: 단순한 문법 암기가 아니라, 객체지향의 4대 특징(캡슐화, 상속, 다형성, 추상화)을 이해하고 이를 통해 '유지보수하기 좋은 코드'를 짤 수 있는지 확인한다.

### **Core Concept (핵심 개념 정리)**

> 단순한 정의가 아닌, 의미 + 구조 + 비교 + 조건 중심으로 작성한다.

**다음 구조를 포함하면 가장 효과적**

|요소|내용|
|---|---|
|**개념 정의**|현실 세계의 사물(Object)을 속성(Field)과 행위(Method)를 가진 프로그래밍 객체로 모델링하여 상호작용하게 만드는 프로그래밍 패러다임.|
|**Java가 OOP인 이유**|1. **클래스 기반**: 모든 코드는 클래스 내부에 존재해야 함.<br>2. **4대 특징 지원**: 캡슐화(접근제어자), 상속(extends), 다형성(Overriding/Interface), 추상화(Abstract)를 언어 차원에서 강제 및 지원함.|
|**동작 방식**|설계도(Class) 작성 → `new` 키워드로 힙 메모리에 실체화(Instance) → 객체 간 메시지(Method Call)를 주고받으며 로직 수행.|
|**장점/단점**|**장점**: 모듈화로 인한 유지보수 용이, 코드 재사용성 증가.<br>**단점**: 초기 설계 비용이 크고, 절차지향 언어(C) 대비 실행 속도가 상대적으로 느릴 수 있음.|
|**예시/비교**|**절차지향(C)**: '무엇을' 할지 순서대로 나열 (함수 중심).<br>**객체지향(Java)**: '누가' 그 일을 할지 정의하고 객체에게 시킴 (객체 중심).|

### **Interview Answer Version (면접 답변식 요약)**

> 30~40초 분량으로, **정의 → 쓰는 이유 → 핵심 효과/특징**순으로 말하듯 요약

"객체지향 언어란 데이터와 로직을 '객체'라는 단위로 묶어 서로 상호작용하도록 설계하는 언어입니다. Java는 이 객체지향의 4대 특징인 **캡슐화, 상속, 다형성, 추상화**를 언어적 차원에서 완벽하게 지원하며, 모든 코드가 클래스 기반으로 작성되기 때문에 대표적인 객체지향 언어라고 할 수 있습니다. 이를 통해 **코드의 재사용성**을 높이고 **유지보수**를 용이하게 하여 대규모 시스템 개발에 적합합니다."

### **Practical Tip (사용시 주의할 점 or 활용 예)**

> 실제로 어떻게 쓰이며, 잘못 쓰면 어떤 문제가 생기는지를 구체적인 예시로 작성한다.

* **흔히 발생하는 오해 (Getter/Setter 남발)**:
    * 객체의 필드를 `private`으로 막고 무조건 Getter/Setter를 다 열어두면 사실상 캡슐화가 깨진 것과 같습니다. 객체는 자신의 상태를 스스로 변경하도록 메서드를 제공해야 합니다.
    * *Bad*: `account.setBalance(account.getBalance() - 1000);` (외부에서 데이터 조작)
    * *Good*: `account.withdraw(1000);` (객체에게 행동을 요청)
* **상속(Inheritance)의 오용**:
    * 코드 재사용만을 위해 상속을 무분별하게 사용하면 부모 클래스와 결합도가 너무 높아집니다.
    * **Tip**: "Is-a 관계(사람은 동물이다)"가 확실하지 않다면 상속보다는 **조합(Composition, Has-a 관계)**을 사용하는 것이 더 유연합니다.
* **스프링 프레임워크 활용**:
    * Spring의 DI(의존성 주입)는 객체지향의 **다형성(Polymorphism)**과 **인터페이스**를 극한으로 활용하여 유연한 구조를 만드는 대표적 사례입니다.
