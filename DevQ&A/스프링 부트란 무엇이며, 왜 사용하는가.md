# 스프링 부트란 무엇이며, 왜 사용하는가? (Spring vs Spring Boot)

스프링은 자바 엔터프라이즈 개발의 혁명이었지만, 한 가지 큰 진입 장벽이 있었습니다. 바로 **"설정(Configuration)이 너무 복잡하다"** 는 것이었습니다.

초기 프로젝트 세팅에만 며칠이 걸리던 시절, 구세주처럼 등장한 것이 바로 **Spring Boot**입니다. 오늘은 스프링 부트가 무엇이고, 왜 현대 자바 백엔드 개발의 표준이 되었는지 정리해 보겠습니다.

---

## 1. Spring Boot란?

Spring Boot는 스프링 프레임워크를 기반으로 한 **"설정을 최소화하고, 바로 실행 가능한(Production-ready) 애플리케이션을 쉽게 만들 수 있게 해주는 도구"** 입니다.

스프링 프레임워크가 "기능"을 제공한다면, 스프링 부트는 그 기능을 "쉽게 사용할 수 있는 환경"을 제공합니다.

> **Spring Boot의 핵심 모토:**
> "Convention over Configuration" (설정보다 관례)

복잡한 설정을 개발자가 일일이 지정하는 대신, 자주 사용되는 설정을 관례적으로 미리 지정해두어 개발자는 **비즈니스 로직에만 집중**할 수 있게 해줍니다.

---

## 2. 왜 Spring Boot를 사용하는가? (핵심 기능 3가지)

스프링 부트를 사용하는 이유는 명확합니다. **개발 생산성**과 **운영 편의성** 때문입니다. 이를 가능하게 하는 3가지 핵심 요소가 있습니다.

### ① 의존성 관리 간소화 (Starter Dependencies)

과거 스프링 프로젝트에서는 웹 개발을 하려면 Spring MVC, Jackson(JSON), Tomcat 등 라이브러리 버전을 하나하나 맞추고 호환성을 체크해야 했습니다. (일명 'Jar 지옥')

스프링 부트는 **`spring-boot-starter`** 라는 묶음 의존성을 제공합니다.

* **예시:** 웹 개발을 하고 싶다면?
    * `build.gradle`에 `implementation 'org.springframework.boot:spring-boot-starter-web'` 한 줄만 추가하면 끝납니다.
    * 관련된 모든 라이브러리(MVC, REST, Tomcat 등)를 검증된 버전으로 알아서 가져옵니다.

### ② 자동 설정 (Auto Configuration)

아마 스프링 부트의 가장 강력한 마법일 것입니다.
`@EnableAutoConfiguration` (또는 `@SpringBootApplication`) 어노테이션을 통해, 스프링 부트는 프로젝트에 추가된 `jar` 파일(의존성)을 보고 **자동으로 필요한 빈(Bean)을 설정**해줍니다.

* **예시:** H2 Database 라이브러리가 클래스 패스에 있다면?
    * 스프링 부트는 "어, H2가 있네? 그럼 인메모리 DB를 쓸 거구나"라고 판단하고, `DataSource` 빈을 자동으로 생성하여 DB 연결을 셋팅해버립니다. 복잡한 XML 설정 없이도 말이죠.

### ③ 내장 WAS (Embedded Server)

기존 스프링 애플리케이션은 코드를 짠 뒤 WAR 파일로 빌드하고, 별도의 WAS(Tomcat, Jetty 등) 서버를 설치한 뒤 그 안에 배포해야 했습니다.

하지만 스프링 부트는 **Tomcat이 내장(Embedded)** 되어 있습니다.
그냥 `main()` 메서드를 실행하기만 하면 서버가 켜집니다. 배포할 때도 별도의 서버 설치 없이 JAR 파일 하나만 실행(`java -jar app.jar`)하면 웹 서비스가 동작합니다. 이는 최근의 클라우드/컨테이너(Docker) 환경과도 찰떡궁합입니다.

## 3. Spring vs Spring Boot 한눈에 보기

| 특징 | Spring Framework (Legacy) | Spring Boot |
| :--- | :--- | :--- |
| **설정** | 매우 복잡 (XML 파티, 긴 Java Config) | 매우 간단 (application.properties/yml) |
| **의존성** | 호환되는 버전을 직접 찾아서 조합 | `Starter`로 검증된 라이브러리 묶음 제공 |
| **실행 환경** | 별도의 외장 WAS(Tomcat) 설치 필요 | 내장 WAS 포함 (JAR 실행 가능) |
| **목표** | 엔터프라이즈 기능 제공 (프레임워크) | 빠른 개발 및 실행 (솔루션) |

---

## 4. 결론

**"Spring Boot makes it easy to create stand-alone, production-grade Spring based Applications that you can 'just run'."**
(스프링 부트는 단독 실행 가능하고, 상용 수준의 스프링 기반 애플리케이션을 쉽게 만들 수 있게 해줍니다. 그냥 실행하세요.) - *Spring Boot 공식 문서*

결론적으로, 스프링 부트는 스프링을 더 이상 어렵게 쓰지 말라는 스프링 팀의 배려이자 강력한 도구입니다. 백엔드 개발자로서 빠른 프로토타이핑과 안정적인 운영 환경 구축을 원한다면 스프링 부트는 선택이 아닌 **필수**입니다.
