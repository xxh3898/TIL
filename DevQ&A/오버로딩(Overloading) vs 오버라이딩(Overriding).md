# 오버로딩(Overloading) vs 오버라이딩(Overriding): 한 끝 차이의 미학

## 1. 개요 (Overview)
이름이 비슷해서 초급 개발자들이 가장 많이 혼동하는 개념입니다. 하지만 **'어디서(Where)', '언제(When)', '어떻게(How)'** 동작하는지 알면 명확히 구분됩니다.

- **오버로딩 (Overloading):** **"과적하다"**. 같은 이름의 메서드에 매개변수만 다르게 하여 여러 개를 만드는 것. (새로운 메서드 정의)
- **오버라이딩 (Overriding):** **"덮어쓰다"**. 부모 클래스에게 물려받은 메서드를 자식 클래스가 재정의하는 것. (기존 메서드 수정)

---

## 2. 상세 개념 및 동작 (Mechanism)

### 2.1 오버로딩 (Overloading) - 확장
- **목적:** 편의성. 같은 기능을 하는 메서드들의 이름을 하나로 통일하기 위함입니다.
- **조건:** 메서드 이름은 **같아야** 하고, 매개변수의 개수나 타입은 **달라야** 합니다. (리턴 타입은 상관없음)
- **예시:** `System.out.println()`이 문자열, 숫자, 불리언을 모두 출력할 수 있는 이유입니다.

### 2.2 오버라이딩 (Overriding) - 재정의
- **목적:** 다형성. 부모가 만든 기능이 마음에 들지 않거나, 자식에게 맞게 변경해야 할 때 사용합니다.
- **조건:** 메서드 이름, 매개변수, 리턴 타입이 모두 **같아야** 합니다.
- **필수:** 상속 관계가 전제되어야 합니다.

---

## 3. 핵심 비교 (Comparison Table)

| 구분 | 오버로딩 (Overloading) | 오버라이딩 (Overriding) |
| :--- | :--- | :--- |
| **우리말 의미** | 적재 (새로 추가) | 재정의 (덮어쓰기) |
| **발생 위치** | 같은 클래스 내 | 상속 관계 (부모-자식 간) |
| **메서드 이름** | 동일 | 동일 |
| **매개변수** | **달라야 함** (개수 or 타입) | **같아야 함** |
| **리턴 타입** | 상관없음 | 같아야 함 (하위 호환 가능) |
| **접근 제어자**| 자유로움 | 부모보다 좁아질 수 없음 |
| **다형성 종류**| **정적 다형성** (Compile-time) | **동적 다형성** (Run-time) |

## 4. 예시 코드 (Example Code)

### 4.1 오버로딩 (Overloading)
```java
class Calculator {
    // 1. 기본 더하기
    int add(int a, int b) {
        return a + b;
    }

    // 2. 오버로딩 (매개변수 타입이 다름)
    double add(double a, double b) {
        return a + b;
    }

    // 3. 오버로딩 (매개변수 개수가 다름)
    int add(int a, int b, int c) {
        return a + b + c;
    }
}
```

### 4.2 오버라이딩 (Overriding)
```java
class Animal {
    void cry() {
        System.out.println("동물이 웁니다.");
    }
}

class Cat extends Animal {
    // 부모의 cry()를 재정의 (Overriding)
    @Override // 컴파일러에게 오버라이딩임을 명시 (안전장치)
    void cry() {
        System.out.println("야옹~");
    }
}

class Dog extends Animal {
    @Override
    void cry() {
        System.out.println("멍멍!");
    }
}
```

## 5. 사용 시 주의할 점 (Precautions)

### 5.1 오버로딩 주의사항: 모호성(Ambiguity)
```java
void test(int a, long b) { ... }
void test(long a, int b) { ... }

// 호출 시: test(1, 1); // 에러 발생!
```
컴파일러가 둘 중 어떤 메서드를 호출해야 할지 헷갈리는 경우 컴파일 에러가 발생합니다.

### 5.2 오버라이딩 주의사항: 제약 조건
1.  **접근 제어자:** 부모 메서드가 `protected`라면, 자식은 `protected`나 `public`만 가능합니다. (더 좁게 변경 불가)
2.  **예외(Exception):** 부모 메서드보다 더 많은(상위의) 예외를 던질 수 없습니다.
3.  **final 메서드:** 부모 메서드에 `final`이 붙어있으면 오버라이딩이 금지됩니다.

---

## 6. 결론 (Conclusion)

> **"오버로딩은 새로운 메서드를 정의하는 것(New)이고, 오버라이딩은 상속받은 메서드를 변경하는 것(Change)입니다. 오버로딩은 컴파일 시점에 결정되지만, 오버라이딩은 런타임에 결정되어 다형성의 핵심이 됩니다."**

이 개념은 자바뿐만 아니라 모든 객체지향 언어의 공통 사항이므로 확실히 익혀두는 것이 좋습니다.
