## Topic (자바의 실행 방식) 2025-11-29

- **주제:** 자바는 컴파일 언어인가, 인터프리터 언어인가? (하이브리드 시스템의 이해)

### **Why (왜 사용하는가? 왜 중요한가?)**

> 이 개념이 실무, 설계, 면접에서 중요한 이유를 정리한다.

- **실무**: 빌드 과정(컴파일)과 런타임 환경(인터프리터/JIT)을 구분해야 배포 시 발생하는 에러와 실행 시 발생하는 에러를 명확히 구별할 수 있다.

- **구조적 의미**: **"Write Once, Run Anywhere"**를 실현하기 위해 OS에 종속되지 않는 바이트코드를 생성하고, 실행 시점에 이를 해석하는 하이브리드 방식을 취한다.

- **면접 의도**: 자바의 실행 메커니즘을 정확히 이해하고 있는지, C언어(컴파일)나 파이썬(인터프리터)과의 차이점을 명확히 아는지 확인한다.

### **Core Concept (핵심 개념 정리)**

> 단순한 정의가 아닌, 의미 + 구조 + 비교 + 조건 중심으로 작성한다.

### **Interview Answer Version (면접 답변식 요약)**

> 30~40초 분량으로, **정의 → 쓰는 이유 → 핵심 효과/특징**순으로 말하듯 요약

"자바는 컴파일 언어와 인터프리터 언어의 특징을 결합한 **하이브리드 언어**입니다. 소스 코드를 먼저 기계가 아닌 가상 머신이 이해할 수 있는 **바이트코드**로 컴파일하여 플랫폼 독립성을 확보하고, 실행 시점에 JVM이 이를 OS에 맞는 기계어로 해석하여 실행합니다. 성능 문제를 보완하기 위해 실행 중 자주 쓰이는 코드를 기계어로 변환해두는 **JIT 컴파일러**를 사용합니다."

### **Practical Tip (사용시 주의할 점 or 활용 예)**

> 실제로 어떻게 쓰이며, 잘못 쓰면 어떤 문제가 생기는지를 구체적인 예시로 작성헌다.

- **컴파일 에러 vs 런타임 에러**:
    
    - IDE에서 빨간 줄이 뜨거나 빌드가 안 되는 건 **컴파일 에러** (문법 오류, 타입 불일치).
    
    - 빌드는 됐는데 실행 중 죽는 건 **런타임 에러** (0으로 나누기, NullPointerException). 이 둘을 명확히 구분해야 디버깅이 빠름.
    
- **Warm-up 이슈**:
    
    - 애플리케이션 시작 직후에는 JIT 최적화가 덜 되어 속도가 느릴 수 있음. 대용량 트래픽을 받는 서버라면 배포 직후 웜업(Warm-up) 과정을 고려해야 함.
