# 프레임워크(Framework): 개발자를 위한 반조리 키트

## 1. 개요 (Overview)
### 1.1 정의
**프레임워크(Framework)**는 단어 그대로 **'뼈대(Frame)'**를 미리 만들어 놓고, 그 안에서 **'일(Work)'**을 할 수 있도록 제공되는 소프트웨어 환경입니다.
애플리케이션을 개발할 때 필요한 기본적인 구조, 코드, 라이브러리, 규칙 등을 모두 제공해주기 때문에 개발자는 **핵심 비즈니스 로직(기능 구현)**에만 집중할 수 있습니다.

### 1.2 비유: "밀키트(Meal Kit)" vs "직접 요리"
- **직접 요리 (No Framework):** 밭에서 파를 뽑고, 소를 잡아 고기를 다지고, 육수를 3시간 끓여야 함. (자유도는 높지만 힘들고 오래 걸림)
- **프레임워크 (Framework):** 손질된 채소, 양념장, 고기, 조리 설명서가 들어있는 **밀키트**. 설명서(규칙)대로 끓이기만 하면 됨. (자유도는 낮지만 빠르고 맛이 보장됨)

---

## 2. 라이브러리(Library)와의 차이점
많은 초심자가 라이브러리와 프레임워크를 혼동합니다. 이 둘을 구분하는 결정적인 차이는 **"누가 주도권을 쥐고 있는가(제어의 흐름)"**입니다.



| 구분 | 라이브러리 (Library) | 프레임워크 (Framework) |
| :--- | :--- | :--- |
| **주도권** | **개발자 (User)** | **프레임워크 (Framework)** |
| **동작 방식** | 내가 필요할 때 도구를 가져다 씀 | 프레임워크가 내 코드를 호출해서 씀 |
| **비유** | 망치, 톱 (내가 씀) | 자동차 생산 라인, 밀키트 (정해진 틀에 내가 맞춤) |
| **핵심 용어** | 제어 (Control) | **제어의 역전 (IoC - Inversion of Control)** |

> **제어의 역전(IoC):**
> 원래는 개발자가 모든 흐름을 제어해야 하지만, 프레임워크를 쓰면 프레임워크가 전체 흐름을 쥐고 있고 개발자가 작성한 코드를 "필요할 때 호출"해줍니다. (예: 스프링이 알아서 내 Controller를 실행시킴)

---

## 3. 프레임워크를 사용하는 이유 (Why?)
1.  **생산성 향상:** 바닥부터 만들 필요가 없어 개발 속도가 매우 빠릅니다.
2.  **코드 품질 보장:** 검증된 아키텍처와 디자인 패턴이 적용되어 있어, 초급 개발자가 짜도 일정 수준 이상의 코드가 나옵니다.
3.  **유지보수 용이:** 팀원 모두가 같은 규칙(구조)으로 코드를 짜기 때문에, 남의 코드를 봐도 이해하기 쉽습니다.

---

## 4. 대표적인 종류 (Examples)

### 4.1 Java
-   **Spring Framework (Spring Boot):** 한국 백엔드 개발의 표준. 엔터프라이즈급(대기업) 웹 애플리케이션 개발에 최적화되어 있습니다.

### 4.2 Python
-   **Django:** "건전지 포함(Batteries Included)"이라는 철학처럼, 웹 개발에 필요한 모든 기능(관리자 페이지, DB 등)이 다 들어있습니다.
-   **FastAPI / Flask:** 좀 더 가볍고 자유도가 높은 프레임워크입니다.

### 4.3 JavaScript (Frontend)
-   **Angular, Vue.js:** 프레임워크의 성격이 강합니다.
-   **React:** 엄밀히 말하면 '라이브러리'지만, 생태계 전체를 묶어 프레임워크처럼 사용합니다.

---

## 5. 단점 (Cons)
1.  **높은 러닝 커브:** 프레임워크가 정해준 규칙(사용법)을 배우는 데 시간이 걸립니다. (예: 스프링의 DI, AOP 개념 익히기)
2.  **제약 사항:** 프레임워크가 제공하는 틀을 벗어나서 개발하려면 오히려 더 복잡해질 수 있습니다.
3.  **무거움:** 사용하지 않는 기능까지 포함되어 있어 프로젝트 용량이 커질 수 있습니다.

---

## 6. 결론 (Conclusion)
프레임워크는 **"개발자를 위한 반제품 집 짓기 세트"**입니다.

맨땅에 헤딩하지 않게 도와주지만, 그 세트의 조립 설명서를 꼼꼼히 읽어야 집을 지을 수 있습니다. 현재 공부하시는 **Spring Boot**가 바로 자바 언어로 웹을 만들기 위한 가장 강력한 집 짓기 세트라고 보시면 됩니다.
