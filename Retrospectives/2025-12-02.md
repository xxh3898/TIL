## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **개인 프로젝트 환경 설정 (Cube & Board)**
    - 큐브 관련 기능(공식, 타이머)과 게시판 기능을 포함하는 리액트 프로젝트 구조를 잡았다.
    - 폴더 구조: `pages`(페이지 단위), `components`(재사용 컴포넌트), `routes`(라우팅 설정)로 분리하여 체계적으로 관리.

- **React Router 구조화**
    - `BrowserRouter`와 `Routes`를 사용하여 SPA(Single Page Application) 라우팅을 구현했다.
    - 중첩 라우팅(Nested Routes)을 활용하여 `Layout` 컴포넌트가 모든 하위 페이지(`Home`, `Board`, `Timer` 등)를 감싸도록 설정했다.

- **Layout 컴포넌트와 Outlet**
    - 헤더(Header)와 같은 공통 요소를 매 페이지마다 작성하지 않고, `Layout.jsx`에 한 번만 작성한 뒤 `Outlet` 컴포넌트를 통해 내부 콘텐츠만 바뀌도록 구현했다.

- **페이지 이동과 현재 위치 감지**
    - `Link` 컴포넌트로 페이지 새로고침 없는 이동을 구현.
    - `useLocation` 훅을 사용하여 현재 사용자가 어떤 경로(Pathname)에 있는지 콘솔로 확인하는 기능을 넣어보았다.

## 🧑🏻‍💻 코드 예제
> 수업 시간에 작성한 핵심 코드나 직접 변형한 코드

```jsx
// 1. 중첩 라우팅을 이용한 레이아웃 적용 (routes.jsx)
const AppRoutes = () => {
    return (
        <BrowserRouter>
            <Routes>
                {/* Layout 컴포넌트가 하위 Route들을 감쌈 */}
                <Route element={<Layout />}>
                    <Route path="/" element={<Home />} />
                    <Route path="/board" element={<BoardIndex />} />
                    <Route path="/algorithms" element={<Algorithms />} />
                    <Route path="/timer" element={<Timer />} />
                    <Route path="/login" element={<Login />} />
                    <Route path="/*" element={<NotFound />} />
                </Route>
            </Routes>
        </BrowserRouter>
    )
}
```

```jsx
// 2. 공통 레이아웃 컴포넌트 (Layout.jsx)
import { Outlet } from 'react-router-dom'

const Layout = () => {
    return (
        <LayoutContainer>
            <Header /> {/* 모든 페이지에 고정 */}
            <MainContent>
                <Outlet /> {/* URL에 따라 바뀌는 부분 */}
            </MainContent>
        </LayoutContainer>
    )
}
```

## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: 라우터를 설정하고 페이지를 이동했는데 화면이 전체가 리렌더링(새로고침) 되는 현상이 발생할 뻔함.
- **해결**: HTML의 `<a>` 태그 대신 `react-router-dom`의 `<Link>` 또는 `<NavLink>`를 사용하여 브라우저 주소만 바꾸고 JS로 화면을 갈아 끼우는 SPA 방식으로 동작하게 함.
