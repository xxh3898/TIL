## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **React Router (SPA Routing)**
    - 기존의 `<a>` 태그를 사용한 페이지 이동은 서버에 새로운 요청을 보내고 화면이 깜빡이는 문제가 있다.
    - `react-router-dom`을 사용하면 브라우저 주소만 변경하고, 필요한 컴포넌트만 갈아 끼우는 **SPA(Single Page Application)** 방식의 라우팅이 가능하다.

- **핵심 컴포넌트**
    - `BrowserRouter`: HTML5 History API를 사용하여 주소 변경을 감지하고 관리하는 최상위 컴포넌트.
    - `Routes` & `Route`: URL 경로(`path`)와 보여줄 컴포넌트(`element`)를 매핑하는 규칙을 정의한다.
    - `Link`: `<a>` 태그 대신 사용하여 페이지를 리로드하지 않고 경로만 변경한다.

- **동적 라우팅 (Dynamic Routing)**
    - URL에 변수를 담아 전달하는 방식.
    - **URL 파라미터**: `/profile/:username` 처럼 경로에 값을 포함. `useParams` 훅으로 값을 받아온다.
    - **쿼리 스트링**: `?sort=asc` 처럼 key-value 쌍으로 데이터 전달. `useLocation` 훅으로 전체 쿼리를 받아오거나, `URLSearchParams`를 통해 파싱한다.

- **페이지 이동 훅 (useNavigate)**
    - 버튼 클릭이나 함수 내부에서 특정 로직 수행 후 페이지를 이동시켜야 할 때 사용한다.
    - `Maps("/")` 또는 `Maps(-1)` (뒤로 가기) 등으로 활용 가능.

## 🧑🏻‍💻 코드 예제
> 수업 시간에 작성한 핵심 코드나 직접 변형한 코드

```jsx
// 1. 기본 라우팅 설정 (App.jsx)
import { BrowserRouter, Routes, Route, Link } from 'react-router-dom'

function App() {
  return (
    <BrowserRouter>
      <nav>
        {/* SPA 방식의 이동 */}
        <Link to="/">홈</Link> 
        <Link to="/about">소개</Link>
      </nav>
      <Routes>
        <Route path='/' element={<Home />}/>
        <Route path='/profile/:username' element={<Profile />}/>
        <Route path='*' element={<NotFound />}/> {/* 404 페이지 처리 */}
      </Routes>
    </BrowserRouter>
  )
}
```

```jsx
// 2. 동적 파라미터와 쿼리 스트링 사용 (Profile.jsx)
import { useLocation, useParams } from 'react-router-dom'

const Profile = () => {
    const { username } = useParams(); // URL 파라미터 (/profile/kim) -> kim
    const { search } = useLocation(); // 쿼리 스트링 (?sort=date) -> ?sort=date
    const query = new URLSearchParams(search);
    const sort = query.get('sort'); // date

    return (
        <div>
            <p>{username}님의 프로필 (정렬: {sort})</p>
        </div>
    )
}
```

```jsx
// 3. 프로그래매틱 이동 (NotFound.jsx)
import { useNavigate } from 'react-router-dom';

const NotFound = () => {
    const navigate = useNavigate();

    const goHome = () => {
        alert("홈으로 이동합니다!");
        navigate("/"); // 홈 경로로 강제 이동
    }
    
    return <button onClick={goHome}>홈으로 가기</button>
}
```

## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: `a` 태그를 사용해서 메뉴를 만들었더니 페이지를 이동할 때마다 새로고침이 되면서 상태(State)가 다 초기화되는 현상이 발생함.
- **해결**: React Router의 `Link` 컴포넌트로 교체하여 해결함. `Link`는 내부적으로 `a` 태그를 생성하지만 클릭 이벤트를 가로채서 브라우저의 기본 동작(페이지 리로드)을 막고 History API만 조작하기 때문임.

## 🌟 잘한 점
> 오늘 하루 스스로 칭찬할 만한 점

- 라우팅 처리를 단순히 따라 치는 것에 그치지 않고, `useParams`와 `useLocation`을 활용해 실제 데이터를 URL로 주고받는 흐름을 직접 테스트해본 점.
- `NotFound` 페이지를 `path='*'`로 처리하여 정의되지 않은 경로 접근을 깔끔하게 방어한 점.
