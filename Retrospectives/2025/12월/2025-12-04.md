## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **자바스크립트 비동기 처리 (Asynchronous)**
    - **Callback**: 과거에 비동기를 처리하던 방식. 중첩되면 '콜백 지옥'이 발생해 가독성이 떨어진다.
    - **Promise**: 비동기 작업의 성공/실패 여부를 객체로 반환한다. `then`, `catch` 체이닝을 사용한다.
    - **Async / Await**: Promise를 더 편하게 사용할 수 있는 문법. `await` 키워드를 쓰면 비동기 코드를 동기 코드처럼 순서대로 작성할 수 있어 가독성이 매우 좋다.

- **Axios & API 연동**
    - `fetch` 대신 `axios` 라이브러리를 사용하여 HTTP 요청을 보냈다. JSON 변환 과정이 필요 없고, 에러 처리가 더 직관적이다.

- **Zustand에서 비동기 상태 관리**
    - 스토어 내부의 액션(함수)을 `async`로 정의하여 API 호출을 처리했다.
    - **상태 3단계**: `loading`(로딩 중), `success`(데이터 저장), `error`(에러 발생) 상태를 나누어 UI에 반영하는 패턴을 익혔다.

## 🧑🏻‍💻 코드 예제
> 수업 시간에 작성한 핵심 코드나 직접 변형한 코드

```javascript
// 1. Zustand Store에서 비동기 통신 처리 (postStore.js)
const usePostStore = create((set) => ({
    posts: [],
    loading: false,
    error: null,
    
    getPosts: async () => {
        set({ loading: true, error: null }); // 로딩 시작
        try {
            const response = await axios.get('[https://jsonplaceholder.typicode.com/posts](https://jsonplaceholder.typicode.com/posts)');
            set({ posts: response.data, loading: false }); // 성공 시 데이터 저장
        } catch (error) {
            set({ loading: false, error: error.message }); // 실패 시 에러 저장
        }
    }
}));
```


## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: 게시글 삭제(`deletePost`) 시 서버 응답을 기다리는 동안 사용자가 중복 클릭을 하거나, 어떤 글이 삭제 중인지 알 수 없는 문제가 있었음.
- **해결**: Store의 전역 `loading` 상태만으로는 개별 아이템의 로딩을 표현하기 어려웠다. 컴포넌트 내부에 `deletePostId`라는 로컬 State를 만들어, 삭제 중인 특정 카드 위에만 `LoadingOverlay`를 띄우도록 구현하여 UX를 개선했다.
