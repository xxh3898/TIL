## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **Postman (API 테스트 도구)**
    - 백엔드 API 및 프론트엔드 연동 테스트를 위한 클라이언트 도구.
    - **Collections**: 관련된 API 요청들을 그룹화하여 관리할 수 있다.
    - **Environments**: 개발 서버, 운영 서버 등의 URL을 환경 변수(`{{base_url}}`)로 설정하여 재사용성을 높인다.

- **람다 표현식 (Lambda Expression)**
    - 메서드를 하나의 식(Expression)으로 표현하여 코드를 간결하게 만든다.
    - **함수형 인터페이스**: 추상 메서드가 단 하나만 있는 인터페이스에서만 람다를 사용할 수 있다.
    - 문법: `(매개변수) -> { 실행문 }`

- **Stream API**
    - 데이터의 '저장'이 아닌 **'처리'**에 목적을 둔 도구.
    - `for`문이나 `if`문 없이 선언적으로 데이터를 필터링하고 매핑할 수 있다.
    - **3단계 처리 과정**: 생성하기 -> 중간 연산(가공) -> 최종 연산(결과 도출).

- **메서드 참조 (Method Reference)**
    - 람다식이 단 하나의 메서드만 호출할 때, `클래스명::메서드명`으로 더 줄여 쓸 수 있다.
    - 예: `s -> System.out.println(s)` => `System.out::println`

- **Optional**
    - `NullPointerException` (NPE)를 방지하기 위해 값을 감싸는 래퍼 클래스.
    - 값이 있을 수도 있고 없을 수도 있는 상황을 명시적으로 표현한다.

## 🧑🏻‍💻 코드 예제
> 수업 시간에 작성한 핵심 코드나 직접 변형한 코드

```java
// 1. 람다 표현식 (익명 클래스 대체)
// Before
Runnable r1 = new Runnable() {
    public void run() { System.out.println("Hello"); }
};
// After
Runnable r2 = () -> System.out.println("Hello");

// 2. Stream & 메서드 참조
List<String> names = List.of("김철수", "이영희", "박지성");
List<String> result = names.stream()
    .filter(name -> name.startsWith("김")) // 중간 연산 (필터)
    .map(String::toUpperCase)              // 메서드 참조
    .collect(Collectors.toList());         // 최종 연산

// 3. Optional 활용
Optional<String> opt = Optional.ofNullable(null);
String name = opt.orElse("이름 없음"); // null이면 기본값 반환
```

## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: 리스트 안에 리스트가 있는 중첩 구조(`List<List<String>>`)를 스트림으로 처리하려고 `map`을 썼더니, 스트림 안에 스트림이 들어가는 형태가 되어 처리가 복잡해짐.
- **해결**: `map` 대신 **`flatMap`** 을 사용해야 중첩된 구조를 하나의 평평한 스트림으로 펼칠 수 있다는 점을 배움.

> 부족했던 점
