## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **IT 인프라와 클라우드**
    - **온프레미스(On-Premise)**: 기업이 직접 전산실을 꾸리고 하드웨어부터 소프트웨어까지 모든 것을 관리하는 방식. 초기 비용이 높고 유연성이 떨어진다.
    - **클라우드(Cloud)**: 인터넷을 통해 컴퓨팅 자원(서버, 스토리지 등)을 필요한 만큼 빌려 쓰고, 사용한 만큼 비용을 지불하는 방식(Pay-as-you-go).

- **클라우드 서비스 모델**
    - **IaaS (Infrastructure as a Service)**: 하드웨어(서버, 네트워크)만 빌려주는 형태. OS나 애플리케이션은 사용자가 직접 관리해야 함. (예: AWS EC2)
    - **PaaS (Platform as a Service)**: 개발에 필요한 환경(런타임, DB 등)까지 제공. 코딩에만 집중 가능. (예: Google App Engine)
    - **SaaS (Software as a Service)**: 완성된 소프트웨어를 서비스 형태로 이용. (예: Gmail, Dropbox)

- **AWS 초기 보안 설정**
    - **루트(Root) 계정**: 모든 권한을 가진 슈퍼 관리자 계정. 일상적인 작업에는 절대 사용하지 말고, **MFA(멀티팩터 인증)**를 설정하여 보호해야 한다.
    - **IAM 사용자**: 권한이 제한된 관리자(Admin) 사용자를 별도로 생성하여 실습에 사용해야 한다.
    - **aws-vault**: 액세스 키를 로컬에 평문으로 저장하지 않고 암호화하여 안전하게 관리해주는 도구.

## 🧑🏻‍💻 코드 예제
> 수업 시간에 작성한 핵심 코드나 직접 변형한 코드

```powershell
# 1. AWS CLI 자격 증명 설정 (PowerShell)
aws configure
# AWS Access Key ID: [IAM 사용자 발급 키 입력]
# AWS Secret Access Key: [비밀 키 입력]
# Default region name: ap-northeast-2 (서울 리전)
# Default output format: json
```

```powershell
# 2. aws-vault를 이용한 안전한 프로필 등록
# Windows 환경에서 Chocolatey로 설치 후 진행
choco install aws-vault

# 프로필 추가 (MFA 연동 시 유용)
aws-vault add admin
aws-vault ls  # 등록된 자격 증명 확인
```

## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: AWS 계정을 만들고 루트 계정으로 계속 실습하려 했으나, 보안상 위험하다는 경고를 받음.
- **해결**: IAM 대시보드에서 `admin` 권한을 가진 IAM 사용자를 새로 생성하고, Google Authenticator를 이용해 루트 계정에 MFA(2단계 인증)를 걸어 잠그는 조치를 취함.

- **문제**: 윈도우 터미널에서 `aws` 명령어가 인식이 안 됨.
- **해결**: AWS CLI MSI 설치 파일을 다운로드하여 설치하고, PowerShell을 재시작하여 환경 변수를 적용시킴.
