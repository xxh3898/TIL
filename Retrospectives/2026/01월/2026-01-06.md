## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **IAM (Identity and Access Management)**
    - "누가(Authentication) 무엇을(Authorization) 할 수 있는지"를 제어하는 AWS의 핵심 보안 서비스.
    - **주요 구성 요소**:
        - **사용자(User)**: 실제 사람이나 애플리케이션 (장기 자격 증명).
        - **그룹(Group)**: 사용자들의 집합으로, 권한 관리를 편하게 해준다.
        - **역할(Role)**: EC2나 Lambda 같은 AWS 서비스가 일시적으로 권한을 얻을 때 사용 (임시 자격 증명 사용).
        - **정책(Policy)**: 권한을 정의하는 JSON 문서 (Action, Resource, Effect 등으로 구성).

- **EC2 (Elastic Compute Cloud)**
    - AWS에서 제공하는 크기 조절이 가능한 클라우드 가상 서버.
    - **AMI (Amazon Machine Image)**: 서버를 찍어내는 템플릿(OS, 애플리케이션 정보 포함).
    - **보안 그룹 (Security Group)**: 인스턴스 앞단의 가상 방화벽으로, 인바운드(들어오는 트래픽)와 아웃바운드(나가는 트래픽) 규칙을 설정한다.
    - **EIP (Elastic IP)**: 인스턴스를 중지하고 다시 시작하면 퍼블릭 IP가 바뀌는데, 이를 고정하고 싶을 때 사용하는 고정 IP 주소.

- **EBS (Elastic Block Store)**
    - EC2에 연결하여 사용하는 영구 블록 스토리지 (가상 하드디스크).
    - EC2 인스턴스가 종료되어도 EBS의 데이터는 유지될 수 있다(설정에 따라 다름).
    - 새로 생성한 EBS는 바로 사용할 수 없으며, **연결(Attach) -> 포맷(Format) -> 마운트(Mount)** 과정을 거쳐야 한다.

## 🧑🏻‍💻 코드 예제
> 수업 시간에 작성한 핵심 코드나 직접 변형한 코드

```bash
# 1. EC2에 Apache 웹 서버 설치 및 실행
sudo yum update -y
sudo yum install httpd -y       # Apache 설치
sudo systemctl start httpd      # 서비스 시작
sudo systemctl enable httpd     # 부팅 시 자동 시작 설정
```

```bash
# 2. EBS 볼륨 포맷 및 마운트 실습
lsblk                           # 1. 디스크 장치 확인 (예: xvdf)
sudo mkfs -t ext4 /dev/xvdf     # 2. 파일 시스템 포맷 (ext4)
sudo mkdir -p /ebs/tmp1         # 3. 마운트할 디렉토리 생성
sudo mount /dev/xvdf /ebs/tmp1  # 4. 마운트 수행

# 마운트 확인
df -h
```

## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: EC2 인스턴스를 중지했다가 다시 켰더니, SSH 접속이 안 되고 웹 사이트도 열리지 않음.
- **해결**: EC2는 재부팅 시 퍼블릭 IP가 변경된다는 점을 배웠다. 이를 해결하기 위해 **Elastic IP (EIP)**를 할당하여 고정 IP를 사용하거나, 실습 단계에서는 바뀐 IP를 다시 확인하여 접속해야 한다.

- **문제**: EBS 볼륨을 콘솔에서 생성하고 EC2에 연결(Attach)했는데, 서버 내부에서 파일을 저장하려고 보니 해당 디스크가 보이지 않음.
- **해결**: 물리적으로 선을 연결한 상태일 뿐, 운영체제에서 인식하고 사용하려면 **포맷(`mkfs`)과 마운트(`mount`)** 과정이 필수라는 것을 깨달았다.
