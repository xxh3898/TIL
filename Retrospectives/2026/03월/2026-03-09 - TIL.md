# TIL - 2026-03-09

## 오늘 배운 것

### 개념
- **JUnit 5**: 자바(Java) 생태계에서 가장 널리 사용되는 오픈 소스 단위 테스트(Unit Test) 프레임워크.
- **단위 테스트**: 작성한 비즈니스 로직 코드가 개발자의 의도대로 정확하게 동작하는지 독립적으로 검증하는 과정.
- **주요 어노테이션**: 
	- `@Test`: 해당 메서드가 테스트 메서드임을 선언.
	- `@BeforeEach`: 각 테스트가 실행되기 직전에 공통적으로 실행할 초기화 작업 지정.
	- `@DisplayName`: 테스트 실행 결과창에 출력될 테스트의 이름을 가독성 좋게 지정.

### 기술
- **테스트 디렉토리 구조화**: 실제 코드가 위치한 `src/main/java`의 패키지 경로와 완벽히 동일한 구조로 `src/test/java` 하위에 패키지를 생성하여 관리.
- **단언문(Assertions) 활용**:
	- `assertEquals(기대값, 실제값)`: 실행 결과가 예상한 값과 일치하는지 검증.
	- `assertThrows(예외클래스, 실행로직)`: 특정 상황에서 의도한 예외(Exception)가 제대로 발생하는지 검증.

## 코드 예제

```java
// 실제 비즈니스 로직 코드 (Calculator.java)
package com.example.project.calculator;

public class Calculator {
    
    public int add(int a, int b) {
        return a + b;
    }

    public int divide(int a, int b) {
        if (b == 0) {
            throw new IllegalArgumentException("0으로 나눌 수 없습니다.");
        }
        return a / b;
    }
}
```

```java
// 테스트 코드 (CalculatorTest.java)
package com.example.project.calculator;

import org.junit.jupiter.api.BeforeEach;
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.assertEquals;
import static org.junit.jupiter.api.Assertions.assertThrows;

class CalculatorTest {

    private Calculator calculator;

    // 각 테스트 메서드 실행 전에 매번 호출되어 초기 상태를 설정
    @BeforeEach
    void setUp() {
        calculator = new Calculator();
    }

    @Test
    @DisplayName("두 숫자를 정상적으로 더해야 한다")
    void testAdd() {
        // Given (준비)
        int a = 10;
        int b = 5;

        // When (실행)
        int result = calculator.add(a, b);

        // Then (검증)
        assertEquals(15, result, "10 + 5는 15여야 합니다.");
    }

    @Test
    @DisplayName("0으로 나눌 경우 IllegalArgumentException 예외가 발생해야 한다")
    void testDivideByZero() {
        // Given (준비)
        int a = 10;
        int b = 0;

        // When & Then (실행과 검증)
        IllegalArgumentException exception = assertThrows(
                IllegalArgumentException.class, 
                () -> calculator.divide(a, b)
        );

        assertEquals("0으로 나눌 수 없습니다.", exception.getMessage());
    }
}
```

## 핵심 정리

- 테스트 코드의 패키지 경로는 실제 로직 코드의 패키지 경로와 동일하게 맞추는 것이 표준 관례이다.
- `@BeforeEach`를 활용하면 중복되는 객체 생성이나 초기화 코드를 줄이고 깔끔한 테스트 환경을 유지할 수 있다.
- 정상적인 입력값에 대한 결과 검증(`assertEquals`)뿐만 아니라, 비정상적인 입력 시 예외가 제대로 터지는지(`assertThrows`) 테스트하는 것도 매우 중요하다.
