# TIL - 2026-03-24


## 오늘 배운 것

### 개념
- **통합 테스트 환경 설계**: 상속 구조(`BaseIntegrationTest`)를 통해 테스트 인프라 설정을 한 곳에서 관리하고 효율적으로 재사용하는 방법.
- **Gradle Wrapper 라이프사이클**: 바이너리 저장소(`.jar`)가 손상되었을 때 시스템 레벨의 도구를 사용하여 직접 복구하는 프로세스.

### 기술
- **Spring Boot 3.4+ @ServiceConnection**: 복잡한 `DynamicPropertySource` 설정 없이 어노테이션 하나로 Testcontainers의 연결 정보를 스프링 빈으로 자동 등록하는 기술.
- **전문 컨테이너 클래스(`MySQLContainer`, `RedisContainer`)**: `GenericContainer`보다 타입 안정성과 스프링 자동 설정 연동성이 뛰어난 전용 클래스 활용법.

---

## 코드 예제

```java
// Spring Boot 3.4+ 환경에서 @ServiceConnection을 활용한 컨테이너 주입
@TestConfiguration(proxyBeanMethods = false)
public class TestcontainersConfiguration {
    @Bean
    @ServiceConnection
    MySQLContainer<?> mysqlContainer() {
        return new MySQLContainer<>(DockerImageName.parse("mysql:8.0"));
    }

    @Bean
    @ServiceConnection
    RedisContainer redisContainer() {
        return new RedisContainer(DockerImageName.parse("redis:7.2"));
    }
}
```

---

## 핵심 정리

- `@ServiceConnection`을 활용하면 테스트용 DB 연결 설정을 수동으로 작성할 필요가 없다.
- `gradle-wrapper.jar` 손상 시 `brew install gradle` 후 `gradle wrapper` 명령으로 바이너리를 재생성할 수 있다.
- `RedisContainer` 전용 클래스를 사용해야 스프링의 `RedisConnectionDetails`가 정상적으로 바인딩된다.
