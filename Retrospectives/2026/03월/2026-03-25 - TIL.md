# TIL - 2026-03-25

## 오늘 배운 것

### 개념
- **CI/CD (Continuous Integration / Continuous Deployment)**: 코드 변경 사항을 자동으로 빌드, 테스트, 배포하는 파이프라인의 중요성과 자동화의 가치.
- **Spring Rest Docs**: 테스트 코드 기반의 API 문서화 도구. 테스트 성공 시에만 문서가 생성되므로 코드와 문서 간의 정합성이 보장됨.
- **Git Fast-Forward Merge**: 브랜치 포인터만 이동시켜 히스토리를 일직선으로 유지하는 병합 방식.

### 기술
- **Testcontainers**: Java 테스트 코드 내에서 Docker 컨테이너(MySQL, Redis 등)를 제어하여 실제 환경과 유사한 통합 테스트 환경 구축.
- **GitHub Actions (paths)**: 모노레포 구조에서 특정 디렉토리의 변경사항이 있을 때만 워크플로우를 트리거하는 최적화 기법.
- **Asciidoctor**: `.adoc` 파일을 HTML/PDF 등 다양한 형식으로 변환해주는 도구.

---

## 코드 예제

### GitHub Actions `paths` 필터 설정
```yaml
on:
  push:
    branches: [ "main", "feat/*" ]
    paths:
      - 'backend/**'
      - '.github/workflows/ci.yml'
```

---

## 핵심 정리
- **Spring Rest Docs**를 사용하면 Swagger와 달리 프로덕션 코드를 어노테이션으로 오염시키지 않으면서도, 테스트를 통한 강력한 문서 보장 체계를 가질 수 있다.
- **모노레포** 환경에서는 모든 변경사항에 대해 전체 테스트를 돌리는 것이 비효율적이므로, `paths` 필터링을 통해 필요한 시점에만 CI가 작동하도록 설계해야 한다.
