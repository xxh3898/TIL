# TIL - 2026-03-27


## 오늘 배운 것

### 개념
- **JPA 명명 전략(Naming Strategy)**: 자동 생성되는 제약조건 명칭의 위험성과 서비스 성장에 따른 명시적 명명의 필요성을 이해함.
- **영속성 전이(Persistence Context)**: 도커 볼륨의 데이터 영속성이 스키마 리팩토링 시 미칠 수 있는 영향(FK 충돌 등)을 실습함.

### 기술
- **JPA Enum 매핑**: `@Enumerated(EnumType.STRING)`이 왜 필수인지 다시 체득함 (`ORDINAL`의 순서 변경 취약성 해결).
- **복합 인덱스(Composite Index)**: `@Index(columnList)` 작성 시 성능 최적화를 위한 컬럼 순서 및 DB 컬럼명 직접 명시 방법.
- **WCA API 설계**: 외부 API 연동 시 핵심 식별자(`wcaId`)를 Enum 내부에 미리 확보하는 미래 지향적 설계 방식.

---

## 코드 예제

```java
// WCA 공식 API 연동 및 영속성 안정성을 모두 고려한 Enum 설계
@Getter
@RequiredArgsConstructor
public enum EventType {
    WCA_333("333", "3x3x3 Cube"),
    WCA_333BF("333bf", "3x3x3 Blindfolded");

    private final String wcaId;      // API 연동용 식별자
    private final String description; // UI 표출용
}

// 엔티티 적용
@Enumerated(EnumType.STRING) // ORDINAL 금지!
@Column(name = "event_type", nullable = false, length = 20)
private EventType eventType;
```

## 핵심 정리

- 데이터베이스 스키마는 한 번 굳어지면 바꾸기 어려우므로 초기 설계 시 **명시적인 이름(FK, IDX)**을 부여하는 것이 관리에 유리함.
- **In-memory(테스트)**와 **Local DB(개발)** 환경의 상태를 동기화하기 위해 자주 도커 컨테이너를 초기화하는 습관이 중요함.
