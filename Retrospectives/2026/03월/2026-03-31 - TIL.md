# TIL - 2026-03-31

## 오늘 배운 것

### 개념
- **도메인 내 비즈니스 룰 캡슐화**: 유저의 PB(Personal Best)를 갱신하거나 스크램블을 생성하는 도메인 자체에 검증(Rule)을 응집시킴으로써 외부 서비스 계층에 종속성을 줄이는 설계를 활용해 보았다.
- **테스트 케이스의 엣지 조건 분리**: 무작위성이 강한 스크램블 생성의 동작 신뢰성을 증명하기 위한 엣지(Edge) 케이스 중심의 테스트 작성 요령(예: 이전 회전과 같은 축 반복 방지, 동일 면 회전 연속 방지)을 연습했다.

### 기술
- **Java Random 및 스트림 활용**: 난수 발생 알고리즘과 자료 수집, Enum 처리 활용
- **JPA 관계 데이터 저장 연동**: `Record` 상태에 따른 `User` 엔티티 내 PB 데이터 업데이트 및 JPA `EntityManager` 기반의 Integration Test

## 핵심 정리

- 비즈니스 정책의 세부 제약 조건(큐빙의 스크램블 조건)들을 철저히 검증하고, 이를 반영하기 위해 도메인 레벨에 상태 확인 로직을 적용하는 방식이 오류 없는 서비스 개발에 있어 필수적이다.
- 여러 엔티티(Record와 User PB)의 영향을 고려하는 트랜잭션의 관리를 철저히 해야 하며, 통합 테스트로 이러한 연동을 보장하는 것이 중요하다.
