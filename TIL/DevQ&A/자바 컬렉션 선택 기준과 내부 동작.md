## Topic
자바 컬렉션 선택 기준과 내부 동작

### 1. Why
- **실무**: 컬렉션은 거의 모든 비즈니스 로직에서 사용된다. `List / Set / Map` 중 무엇을 고르고, `ArrayList / LinkedList`, `HashMap`이 어떻게 동작하는지 이해해야 성능과 코드 의미를 함께 맞출 수 있다.
- **구조적 의미**: 컬렉션 선택은 단순 취향이 아니라 데이터 특성을 코드에 표현하는 일이다. 순서가 중요한지, 중복 허용이 필요한지, key-value 구조인지에 따라 적절한 자료구조가 달라진다.
- **면접 의도**: 면접에서는 컬렉션 이름 암기가 아니라 선택 기준, 내부 동작, 그리고 `equals / hashCode`가 해시 기반 컬렉션과 어떻게 연결되는지 보려는 경우가 많다.

### 2. Core Concept
- **개념 정의**: `List`는 순서를 유지하고 중복을 허용하는 컬렉션이고, `Set`은 중복을 허용하지 않는다. `Map`은 key-value 구조로 데이터를 저장한다. `ArrayList`는 배열 기반, `LinkedList`는 노드 연결 기반이다. `HashMap`은 해시 버킷을 활용한 key-value 저장소다.
- **동작 방식**: `ArrayList`는 조회가 빠르고 끝에 추가하는 작업에 유리하지만, 중간 삽입과 삭제 비용이 클 수 있다. `LinkedList`는 노드 연결 구조라 중간 삽입/삭제에 상대적으로 유리할 수 있지만, 실제로는 탐색 비용과 메모리 오버헤드 때문에 일반적인 실무에서는 `ArrayList`가 더 자주 선택된다. `HashMap`은 key의 `hashCode`로 버킷을 찾고, 같은 해시 버킷 안에서는 `equals`로 최종 비교한다.
- **장점 / 단점**: `List`는 순서 처리에 좋고, `Set`은 중복 제거에 직관적이며, `Map`은 빠른 key 기반 조회에 유리하다. 다만 자료구조 의미에 맞지 않게 쓰면 코드 가독성과 성능이 모두 나빠진다.
- **비교 포인트**: 순서가 중요하면 `List`, 중복 제거가 핵심이면 `Set`, key 기반 조회가 필요하면 `Map`이 기본 선택이다. `ArrayList / LinkedList` 비교에서는 이론상 차이보다 실제 접근 패턴을 봐야 하고, 대부분의 애플리케이션 로직에서는 `ArrayList`가 더 현실적인 기본값이다.
- **사용 조건 / 주의점**: `HashMap`, `HashSet`에 넣는 key나 요소는 `equals / hashCode` 계약을 지켜야 한다. mutable 객체를 key로 쓰면 값이 바뀐 뒤 조회가 안 되는 문제가 생길 수 있고, 자료구조 선택은 중복 허용 여부와 조회 방식부터 기준으로 잡아야 한다.

### 3. Interview Answer Version
- 30~40초 답변: 자바 컬렉션은 데이터 특성에 맞춰 선택하는 게 핵심이라고 생각합니다. 순서가 중요하면 `List`, 중복 제거가 필요하면 `Set`, key-value 조회가 필요하면 `Map`을 선택합니다. 구현체로는 보통 `ArrayList`가 기본값이고, `LinkedList`는 이론적으로 삽입/삭제에 강하지만 실무에서는 탐색 비용과 메모리 오버헤드 때문에 상대적으로 덜 쓰입니다. 또 `HashMap`이나 `HashSet`은 `hashCode`로 먼저 위치를 찾고 `equals`로 최종 비교하기 때문에, 이 두 메서드 계약을 지키는 것이 매우 중요합니다.

### 4. Practical Tip
- 실제 적용 사례: 정렬된 화면 목록 데이터는 `List`, 중복 사용자 ID 제거는 `Set`, 회원 ID로 회원 정보를 빠르게 찾는 캐시 구조는 `Map`으로 두는 식이 가장 직관적이다.
- 잘못 쓰면 생기는 문제: 중복 제거가 필요한데 `List`를 계속 순회하거나, mutable 객체를 `HashMap` key로 쓰면 성능 저하와 조회 실패가 동시에 생길 수 있다.
- 프로젝트 연결: 서비스 계층에서 데이터를 모을 때는 먼저 순서, 중복, 조회 방식이 무엇인지 정하고 컬렉션을 선택해야 한다. 무조건 익숙한 컬렉션만 쓰면 코드 의미가 흐려진다.

### 5. 예상 꼬리 질문
1. 실무에서 `LinkedList`가 상대적으로 덜 쓰이는 이유는 무엇인가요?
- 답변: 이론상 중간 삽입과 삭제는 유리하지만, 실제로는 탐색 비용이 크고 노드 객체가 많아 메모리 효율과 CPU 캐시 친화성이 떨어집니다. 그래서 일반적인 애플리케이션에서는 `ArrayList`가 더 나은 기본값인 경우가 많습니다.
2. `HashMap` 충돌이 발생하면 내부적으로 어떻게 처리되나요?
- 답변: 같은 버킷에 여러 엔트리가 모이면 처음에는 연결 리스트 형태로 관리하고, Java 8 이후에는 조건에 따라 트리 구조로 전환해 성능 저하를 완화합니다. 최종적으로는 `equals`로 실제 key 동일성을 확인합니다.
3. `equals`와 `hashCode`가 해시 컬렉션에서 왜 중요한가요?
- 답변: `hashCode`는 어떤 버킷을 볼지 정하는 기준이고, `equals`는 그 안에서 진짜 같은 객체인지 판단하는 기준입니다. 둘의 계약이 깨지면 `HashMap` 조회 실패나 `HashSet` 중복 처리 오류가 생길 수 있습니다.
