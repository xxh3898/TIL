## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- ES6(ECMAScript 2015)
	- ES6이란 2015년에 발표된 자바스크립트의 6번째 버전이며, 공식 명칭은 ECMAScript 2015이다.
	- ES6은 기존 자바스크립트의 불편한 문법과 일관성을 개선하여 사용성과 생산성을 높였다.
	  React 코드 대부분이 ES6 문법으로 작성되어 있다.

- const, let
	- 기존 var(함수 스코프)와 달리 블록 스코프이다
	- var는 자동으로 undefined로 초기화 되지만 let, const는 초기화 전에 접근이 불가하다.(TDZ)
	- var는 재선언이 가능하지만 let과 const는 재선언이 불가하다. 또한 const는 재할당이 불가하다.

- 화살표 함수
	- 화살표 함수는 function 키워드 대신 => 를 사용하는 간결한 함수이다.
	- 호출한 객체 기준으로 this가 결정되는 일반 함후와 달리, 자신만의 this를 갖지 않고 상위 스코프의 this를 그대로 사용한다.
	- 함수의 구현부에 리턴값만 있다면 return키워드와 {}를 생략 가능, 매개변수가 한 개라면 ()도 생략 가능
	- 장점
		- 코드가 짧고 가독성이 좋다.
		- this 헷갈림 문제가 줄어든다.(콜백 함수에서 특히 유리)
		- 함수 표현식이 더 직관적이다.
	- 단점
		- this를 원하는 대로 쓸 수 없어 객체 메서드에서는 부적합하다.
		- 생성자 함수로 사용이 불가하다.

- 템플릿 리터럴
	- 문자열과 변수 결합 시 유용
	- 여러 줄 문자열 처리

- 구조분해할당
	- 객체에서 필요한 값을 바로 변수로 추출할 때 사용
	- React에서는 props라는 객체를 전달하고 매개변수로 내부의 값들을 받을 때 해당 문법을 주로 사용함
	- 배열 구조분해할당
		- 배열 요소를 순서대로 변수에 담아줌

- 스프레드 연산자(...)
	- 배열/객체 복사, 병합, 수정, 나머지 값을 처리 등에 사용됨
	- 불변성 유지에 중요

- 기본 매개변수
	- 함수 호출 시 인자를 지정하지 않았을 경우를 대응한다.
	- null: 개발자가 명시적으로 표현한 빈값
	  undefined: 시스템이 정해주느 빈값(개발자가 선언 후 사용하지 않은 값)

- 클래스
	- 객체를 생성하기 위한 설계도
	- React 클래스형 컴포넌트 기반
	- 데이터(변수)와 기능(함수), 생성자를 포함하고 있다.
		- js에서 생성자는 명확하게 이름을 constructor라고 정해둠

- 모듈
	- export와 import 문법
		- export: 외부로 기능을 내보냄.
		  함수, 변수 전부 가능
		-  default export: 이름 없이 한 개만 보낼 수 있음
		  해당 모듈을 불러와서 사용하는 쪽에서 이름을 자유롭게 만들 수 있음
	- 컴포넌트 분리 시 필수

- Promis와 async/await
	- 비동기 처리 기본기
	- useEffect에서 fetch 처리 시 유용
	- async / await
		- async 함수는 항상 Promise를 반환
		- await는 Promise가 끝날 때까지 기다림
		- 동기 코드처럼 읽기 쉬움

## 🧑🏻‍💻 코드 예제

``` js
// 화살표 함수
const obj = {
  name: "Kim",
  normal() {
	console.log(this.name); // obj
  },
  arrow: () => {
	console.log(this.name); // 상위 스코프 (보통 window / undefined)
  }
};
```
