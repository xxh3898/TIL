## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **클래스형 컴포넌트 (Class Component)**
    - 함수형 컴포넌트 이전에 사용되던 방식으로, `Component` 클래스를 상속받아 생성한다.
    - `render()` 메서드가 필수적이며, 내부에서 JSX를 반환한다.
    - `this.state`를 통해 컴포넌트 내부 상태를 객체 형태로 관리한다.

- **State (상태)**
    - 컴포넌트 내부에서 관리되는 변경 가능한 데이터.
    - `state`가 변경되면 React는 이를 감지하고 자동으로 재렌더링(Re-rendering)을 수행한다.
    - **중요**: 값을 직접 수정하면 안 되고, 반드시 `setState` 함수를 사용하여 불변성을 유지하며 업데이트해야 한다.

- **생명주기 메서드 (Lifecycle Methods)**
    - 컴포넌트가 생성(Mount), 업데이트(Update), 제거(Unmount)되는 시점에 호출되는 메서드들.
    1. **componentDidMount**: 컴포넌트가 화면에 나타난 직후 호출됨 (API 호출, 타이머 설정 등에 사용).
    2. **componentDidUpdate**: props나 state가 변경되어 업데이트된 직후 호출됨.
    3. **componentWillUnmount**: 컴포넌트가 사라지기 직전 호출됨 (타이머 해제 등 정리 작업).
    4. **shouldComponentUpdate**: 리렌더링 여부를 결정하는 메서드로 성능 최적화에 사용됨 (return false 시 렌더링 안 함).

```jsx
// 1. 생명주기 메서드 실습 (LifecycleTest.jsx)
class LifecycleTest extends Component {
    constructor(props){
        super(props)
        this.state = { count: 0 } // State 초기화
    }

    componentDidMount(){
      console.log("마운트 완료: API 호출이나 이벤트 등록은 여기서!");
    }

    componentDidUpdate(prevProps, prevState){
      console.log("업데이트 완료: 이전 값과 비교 가능", prevState);
    }

    componentWillUnmount(){
      console.log("언마운트: 정리 작업 수행");
    }
}
```

## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: `setState`를 실행했는데 값이 바로 변하지 않고 콘솔에 이전 값이 찍히는 현상.
- **원인**: `setState`는 **비동기(Asynchronous)** 로 동작하기 때문에, 호출 직후에는 아직 state가 업데이트되지 않았을 수 있음.
- **해결**: 변경된 값을 바로 사용해야 한다면 `setState`의 두 번째 인자로 콜백 함수를 넘기거나, `componentDidUpdate` 생명주기 메서드 안에서 처리해야 함을 배움.

## ⚠️ 아쉬운 점
> 부족했던 점

- 클래스형 컴포넌트 코드가 함수형에 비해 확실히 길고 복잡하다(this 바인딩 등). `useEffect` 훅이 이 모든 생명주기를 어떻게 대체하는지 아직은 감이 잘 안 온다.
