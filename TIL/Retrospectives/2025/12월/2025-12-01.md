## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **Context API (전역 상태 관리)**
    - 기존에는 부모 -> 자식 -> 손자 순으로 Props를 계속 넘겨주는 'Props Drilling' 문제가 있었다.
    - `createContext`와 `Provider`를 사용하여 앱 전역에서 데이터(Todo 목록, 추가/삭제 함수 등)를 공유하고, 필요한 곳에서 `useContext`로 바로 꺼내 쓸 수 있게 되었다.

- **Local Storage 활용**
    - 새로고침하면 데이터가 날아가는 문제를 해결하기 위해 브라우저의 `localStorage`를 사용했다.
    - `useEffect`를 활용해 `todos` 상태가 변할 때마다 저장하고, 초기값을 불러올 때 `getItem`으로 가져오는 로직을 구현했다.

- **Outlet & Layout**
    - `react-router-dom`의 `Outlet` 컴포넌트를 활용하여 헤더(Header)는 고정하고 메인 콘텐츠 영역만 바뀌는 레이아웃 구조를 잡았다.

- **필터링 기능 구현**
    - 상태값(all, completed, pending)에 따라 `filter()` 함수를 사용하여 보여줄 리스트를 동적으로 변경하는 로직을 작성했다.

## 🧑🏻‍💻 코드 예제
> 수업 시간에 작성한 핵심 코드나 직접 변형한 코드

```jsx
// 1. Context API로 투두 데이터 및 로직 관리 (TodoContext.jsx)
export const TodoProvider = ({children}) => {
    // 로컬 스토리지에서 초기값 불러오기
    const [todos, setTodos] = useState(() => {
        const savedTodos = localStorage.getItem('todos');
        return savedTodos ? JSON.parse(savedTodos) : [];
    });

    // 투두가 변경될 때마다 로컬 스토리지 저장
    useEffect(() => {
        localStorage.setItem('todos', JSON.stringify(todos));
    }, [todos]);

    const addTodo = (text, category) => { /* ... */ }
    const deleteTodo = (id) => { /* ... */ }
    
    // Provider를 통해 하위 컴포넌트에 값 전달
    return (
        <TodoContext.Provider value={{ todos, addTodo, deleteTodo }}>
            {children}
        </TodoContext.Provider>
    )
}
```

```jsx
// 2. 필터링 로직 구현 (TodoListPage.jsx)
const TodoListPage = () => {
    const { todos } = useTodos();
    const [filter, setFilter] = useState('all');

    // 필터 상태에 따라 보여줄 리스트 계산
    const filteredTodos = todos.filter(todo => {
        if(filter === 'completed') return todo.completed;
        else if(filter === 'pending') return !todo.completed;
        return true;
    });

    return (
        // ... 버튼 UI 생략
        <TodoListContainer>
            {filteredTodos.map(todo => <TodoItem key={todo.id} todo={todo} />)}
        </TodoListContainer>
    )
}
```

## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: `Header` 컴포넌트에서 현재 페이지가 활성화되었는지 표시하는 스타일이 적용되지 않음.
- **해결**: `react-router-dom`의 `useLocation` 훅을 사용하여 현재 경로(`pathname`)를 알아내고, 링크의 경로와 비교하여 `active` 클래스를 조건부로 부여함.

- **문제**: 새로고침 시 화면이 깜빡이거나 데이터가 초기화되는 현상.
- **해결**: `useState`의 초기값으로 함수를 전달하는 **Lazy Initialization** 방식을 사용하여, 컴포넌트가 처음 렌더링될 때만 `localStorage`에 접근하도록 최적화함.
