## 📚 학습 내용
> 오늘 공부하면서 새롭게 배운 내용이나 정리한 개념

- **AWS CLI/콘솔 vs EC2 접속**
    - **AWS 콘솔/CLI**: AWS 리소스(인스턴스 생성, 중지, 삭제 등)를 관리하는 영역.
    - **EC2 접속**: 가상 컴퓨터 내부(OS)로 로그인하여 애플리케이션을 설치하거나 설정을 변경하는 작업.

- **EC2 접속 방법**
    - **SSH 클라이언트**: 터미널에서 `ssh` 명령어를 사용하여 접속. (Mac/Linux는 PEM 키, Windows Putty는 PPK 키 사용)
    - **EC2 Instance Connect**: 별도 키 파일 없이 AWS 웹 콘솔에서 브라우저 기반으로 바로 접속하는 방법 (임시 키 사용).

- **인스턴스 정보 확인 (Metadata & Userdata)**
    - **메타데이터 (Metadata)**: 실행 중인 인스턴스 자체의 정보(IP 주소, 인스턴스 ID, AMI ID 등).
        - 내부에서 `http://169.254.169.254/latest/meta-data/`로 요청하여 조회 가능.
    - **유저데이터 (Userdata)**: 인스턴스가 **최초 실행될 때(부팅 시)** 한 번만 실행되는 스크립트. 패키지 자동 설치나 초기 설정에 사용됨.

- **파일 전송 (FileZilla)**
    - CLI 환경이 낯설 때, SFTP 프로토콜을 지원하는 GUI 도구인 FileZilla를 이용하면 드래그 앤 드롭으로 파일을 쉽게 업로드/다운로드할 수 있다.

## 🧑🏻‍💻 코드 예제
> 수업 시간에 작성한 핵심 코드나 직접 변형한 코드

```bash
# 1. SSH 접속 시 권한 설정 및 접속 명령어
# 키 파일 권한을 '나만 읽기 가능'하게 변경 (필수)
chmod 400 my-key.pem

# SSH 접속 (Amazon Linux 기본 유저는 ec2-user)
ssh -i "my-key.pem" ec2-user@13.125.10.123
```

```bash
# 2. 인스턴스 내부에서 메타데이터 조회
# 인스턴스 ID 확인
curl [http://169.254.169.254/latest/meta-data/instance-id](http://169.254.169.254/latest/meta-data/instance-id)

# 퍼블릭 IP 확인
curl [http://169.254.169.254/latest/meta-data/public-ipv4](http://169.254.169.254/latest/meta-data/public-ipv4)
```

```bash
# 3. 유저데이터(User Data) 스크립트 예시
#!/bin/bash
yum update -y
yum install -y httpd
systemctl start httpd
systemctl enable httpd
echo "<h1>Hello World from $(hostname -f)</h1>" > /var/www/html/index.html
```

## 💣 Trouble Shooting
> 발생한 문제와 해결책

- **문제**: `ssh` 명령어로 접속을 시도했는데 `WARNING: UNPROTECTED PRIVATE KEY FILE!` 에러가 뜨면서 접속이 거부됨.
- **해결**: 리눅스/유닉스 환경에서는 개인 키 파일(.pem)의 권한이 너무 개방되어 있으면 보안상 접속을 막는다. `chmod 400 키파일명.pem` 명령어로 소유자만 읽을 수 있도록 권한을 수정하여 해결했다.

- **문제**: FileZilla로 접속하려는데 `Permission denied` 오류 발생.
- **해결**: 호스트 앞에 `sftp://`를 붙이지 않았거나, 키 파일을 올바르게 등록하지 않아서 발생함. 사이트 관리자 설정에서 로그온 유형을 '키 파일'로 변경하고 PEM 키를 등록하니 정상 접속됨.

## 🌟 잘한 점
> 오늘 하루 스스로 칭찬할 만한 점

- 단순히 서버에 접속하는 것을 넘어, 메타데이터 URL(`169.254.169.254`)을 활용해 서버 스스로 자신의 정보를 가져오게 하는 방법을 실습한 점.
- 유저데이터를 활용해 서버가 켜지자마자 웹 서버가 자동으로 설치되도록 자동화한 점.

## ⚠️ 아쉬운 점
> 부족했던 점

- 유저데이터 스크립트를 작성할 때 오타가 나면 디버깅하기가 어렵다. `/var/log/cloud-init-output.log` 로그 파일을 확인하면 된다고 하는데, 아직 로그 분석이 익숙지 않다.
