# TIL - 2026-03-11

## 오늘 배운 것

### 개념
- **k6**는 오픈소스 성능 테스트 도구로, API나 웹 서비스의 **부하 테스트(load testing)**와 **성능 테스트(performance testing)**를 수행하기 위해 사용된다.
- JavaScript 기반 스크립트로 테스트 시나리오를 작성하며, 가상 사용자(Virtual Users, VU)를 생성해 실제 사용자 트래픽과 유사한 상황을 시뮬레이션한다.
- 주요 목적
	- 시스템의 **최대 처리량(throughput)** 확인
	- **응답 시간(latency)** 측정
	- **병목 지점(bottleneck)** 탐지
	- **장애 발생 시점** 파악
- 테스트 실행 시 k6는 다음과 같은 주요 지표를 제공한다.
	- `http_req_duration`: HTTP 요청 전체 처리 시간
	- `http_reqs`: 총 요청 수
	- `vus`: 동시에 실행 중인 가상 사용자 수
	- `iterations`: 테스트 루프 반복 횟수
	- `checks`: 검증 성공/실패 여부

---

### 기술
- **JavaScript 기반 테스트 스크립트 작성**
- CLI(Command Line Interface)로 테스트 실행
- 다양한 **부하 시나리오 구성**
	- 고정 사용자 수
	- 점진적 증가
	- 스파이크 트래픽
- 테스트 결과는 콘솔 출력 또는 **Grafana 대시보드 연동**으로 시각화 가능
- CI/CD 환경에서도 실행 가능하여 **성능 회귀 테스트**에 활용 가능

---

## 코드 예제

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';

// 테스트 옵션 (부하 단계 설정)
export const options = {
    stages: [
        { duration: '30s', target: 50 }, // 30초 동안 50명의 가상 유저(VUs)로 증가
        { duration: '1m', target: 50 },  // 1분 동안 50명 유지
        { duration: '30s', target: 0 },  // 30초 동안 0명으로 감소
    ],
    thresholds: {
        http_req_duration: ['p(95)<500'], // 95%의 요청이 500ms 이내에 완료되어야 함
        http_req_failed: ['rate<0.01'],   // 에러율이 1% 미만이어야 함
    },
};

// URL 설정 (환경 변수로 덮어쓰기 가능)
const BASE_URL = __ENV.K6_BASE_URL || 'http://localhost:8080';

// 테스트용 더미 계정 정보 (data.sql 기반)
const TEST_EMAIL = __ENV.K6_TEST_EMAIL || 'user4@test.com'; 
const TEST_PASSWORD = __ENV.K6_TEST_PASSWORD || 'pass1234!';

export function setup() {
    // 1. 단 1번 실행되는 setup(): 로그인하여 토큰 획득
    const loginPayload = JSON.stringify({
        email: TEST_EMAIL,
        password: TEST_PASSWORD,
    });

    const loginHeaders = {
        headers: {
            'Content-Type': 'application/json',
        },
    };

    const loginRes = http.post(`${BASE_URL}/api/auth/login`, loginPayload, loginHeaders);

    check(loginRes, {
        'login successful': (r) => r.status === 200,
    });

    // 응답에서 accessToken 속성 추출 
    // (만약 응답 형식이 다르면 r.json('token') 등으로 변경해야 함)
    let token = loginRes.json('accessToken') || loginRes.json('token');

    // Spring Security JWT 필터용 토큰 prefix 추가
    if (token && !token.startsWith('Bearer ')) {
        token = `Bearer ${token}`;
    }

    return { token: token };
}

export default function (data) {
    if (!data.token) {
        console.error('로그인 실패로 토큰이 없습니다. 테스트를 종료합니다.');
        return;
    }

    const params = {
        headers: {
            'Authorization': data.token,
            'Content-Type': 'application/json',
        },
    };

    // 2. 가상 유저(VUs)들이 반복적으로 호출할 메인 타겟 API
    // N+1 문제 해결 (Fetch Join 등) 성능 개선을 명확히 측정할 수 있는 멤버 목록 조회 API
    const res = http.get(`${BASE_URL}/api/admin/team/members?page=0&size=50`, params);

    check(res, {
        'members list status is 200': (r) => r.status === 200,
    });

    // 다음 요청 전 1초 대기하여 적정 부하 유지
    sleep(1);
}

```

설명
1. `vus` → 동시에 실행되는 가상 사용자 수
2. `duration` → 테스트 실행 시간
3. `http.get()` → HTTP 요청 수행
4. `check()` → 응답 검증
5. `sleep()` → 사용자 행동 간 대기 시간 시뮬레이션

## 핵심 정리

- **k6 = JavaScript 기반 성능 테스트 도구**
- 가상 사용자(VU)를 생성해 실제 사용자 트래픽을 시뮬레이션
- API 응답 시간, 처리량, 오류율 등 **핵심 성능 지표 측정**
- CLI 기반이라 **자동화 및 CI/CD 파이프라인에 통합 가능**
- **Grafana와 연동하면 실시간 성능 시각화 가능**

## 추가 학습

- k6 부하 테스트 시나리오 유형
	- **Load Test**: 일반적인 트래픽 상황 테스트
	- **Stress Test**: 시스템 한계 확인
	- **Spike Test**: 갑작스러운 트래픽 증가
	- **Soak Test**: 장시간 안정성 테스트

- k6 성능 테스트 설계 시 고려 사항
	- 실제 사용자 패턴 기반 시나리오 작성
	- 응답 시간 SLA 기준 정의
	- 에러율 임계값 설정
	- 서버 CPU / 메모리 / DB 부하 모니터링 병행
