# TIL - 2026-03-26


## 오늘 배운 것

### 개념
- **Metrics vs Metadata**: 프로메테우스는 숫자 지표(Metrics)를 수집하여 시각화하고, Actuator의 다른 엔드포인트(mappings, env 등)는 정적인 설정 정보(Metadata)를 제공함.
- **Actuator HATEOAS templated**: `templated: true`는 변수가 포함된 URI 템플릿임을 의미하며, 사용자가 값을 채워넣어야 완성된 주소가 됨.

### 기술
- **Spring Boot Docker Compose Support**: `spring.docker.compose.file`과 `working-directory` 속성을 조정하여 프로젝트 루트의 compose 파일을 제어하거나, CI 환경에서의 비생산성(리소스 낭비)을 막는 법.
- **Gradle `--no-daemon`**: GitHub Actions와 같은 일회성 러너(Ephemeral Runner) 환경에서는 자원 효율과 프로세스 안정성을 위해 데몬을 끄는 것이 표준 관례임.

---

## 코드 예제

```yaml
spring:
  docker:
    compose:
      file: ../docker-compose.yml
      working-directory: ..

management:
  endpoints:
    web:
      exposure:
        include: prometheus, health, info
```

---

## 핵심 정리

- 모니터링은 "숫자"의 변화를 추적하고, API 명세(Rest Docs)는 "사용법"을 정의한다.
- CI 파이프라인에서는 테스트 통과뿐만 아니라, 실제 문서 조립(Asciidoctor build) 성공까지 확인해야 장기적인 빌드 안정성이 확보된다.
